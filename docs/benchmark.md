# 知识树性能基准测试报告

## 测试环境

- CPU: Intel Xeon (aliyun)
- 编码器: sentence-transformers all-MiniLM-L6-v2 (384维)
- 推理模型: qwen2.5:7b (Ollama, CPU推理)
- 树存储: 文件系统 (smart_tree/)
- 树规模: 570 节点, 10 层深度

## 当前性能 (570节点)

| 操作 | 耗时 | 说明 |
|------|:----:|------|
| 树加载 | 0.29s | 文件系统 → 内存 |
| 编码器首次加载 | ~6s | 模型下载+加载（一次性） |
| 编码器预热后 | ~0.01s | 384维文本编码 |
| 渗透检索 | **~0.008s** | 10层遍历+相似度计算 |
| Ollama qwen2.5:7b 推理 | ~8s | 7B模型CPU推理（主要瓶颈） |
| 完整链路 | ~8s | 树检索 ≈ 0.1% 总时间 |

### 渗透算法扩展性

渗透不遍历全部节点，只检查每层幸存者的直接子节点：

```
每层: 计算相似度 → 淘汰 → 幸存者(3~8个) → 进入下一层
       |-- 只检查 ~160 个子节点, 不是全部节点 --|
```

总耗时 ≈ 深度 × (幸存者 × 子节点数) × 向量点积时间

## 世界规模预估 (1000万节点)

### 树结构假设

| 指标 | 当前 | 世界规模 |
|------|:----:|:--------:|
| 总节点数 | 570 | 10,000,000 |
| 深度 | 10 | 20 |
| 上层分支数 | 3~5 | 50~100 |
| 下层分支数 | 2~3 | 5~10 |
| 每层检查 | ~3 | ~160 |
| 总向量运算 | ~25 | ~3,200 |

### 响应速度预估

| 阶段 | 耗时 | 瓶颈？ |
|------|:----:|:------:|
| 树加载 (SSD→内存) | ~8-10s | ✅ 一次性 |
| 编码器加载 | ~6s | ✅ 一次性 |
| **渗透检索** | **~150ms** | ❌ **非瓶颈** |
| LLM推理 (同当前) | ~8s | ⚠️ 主要瓶颈 |

**关键结论**: 渗透检索时间从 8ms → **~150ms** (即使增大 17500 倍规模)，仍然远快于 LLM 推理时间。

### 存储需求

| 项目 | 大小 |
|------|:----:|
| 向量 (384维 float32) | 14.3 GB |
| 元数据 (JSON) | ~2 GB |
| 合计 | ~16 GB |
| 加载时间 (SSD 2GB/s) | ~8s |

## 瓶颈分析

```
树加载 ─── 编码器 ─── 渗透检索 ─── LLM推理
 0.3s      6s        0.008s        8s
                        ↓ 扩展到 1000 万节点
                       0.15s        8s
```

- **渗透检索**：即使规模扩大 17500 倍，仍只需 ~150ms
- **LLM推理**：始终是主要瓶颈（~8s），与树规模无关
- **优化方向**：使用更小的 LLM（qwen2.5:0.5b 可降至 ~1s）或 GPU 推理
